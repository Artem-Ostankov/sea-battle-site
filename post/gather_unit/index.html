Gather unit

to gather or not to gather

A 2 minute read, Posted on Wed, Mar 1, 2017

Gather Unit

So for now we go with simple solution. One Gather unit is increasing gold income by 10 gold per turn.

Interesting part is the next one. Gather unit is basically is just a holder of the Gather action. So income is being just another atomic action that changes game state. And therefore it can have its own execution order. Which in turn makes it a bit more complicated and interesting in the same time. Execution order would basically mean that it could for example be before, or after cannon shot, which means you would destroy someones Gather unit BEFORE it will bring money to the player >:-} Not sure this would be the best for players, but by setting it up this way I make it flexible and can play around with execution order when it will come to fine-tuning balance of the game.

For now, to leave things simple and more intuitive, gather execution will be coming first.

Also, to make it as same as other actions, it will start effecting only next turn after unit is built. Which might feel a bit un-intuitive, but would be more homogeneous. Off course, I will need to test it with real people, so if you guys think that it should act directly after build, pls let me know. So again 2 scenarios: Scenario 1: Turn 0: Order Build a Gather Unit Turn 1: It’s built and it will register Gather action at the end of player’s turn. So it has increase income, BUT NOT increases money yet. (so if it was destroyed right after building -> no money) Turn 2: It brings Money. Scenario 2: Turn 0: Order Build a Gather Unit Turn 1: It’s built and it will register Gather action right away and executes it. So it has increase income, AND increases money. Turn 2: It brings Money.

I like to go with Scenario 1 for a mean time, but if you don’t like it and prefer Scenario 2 - ping me.
